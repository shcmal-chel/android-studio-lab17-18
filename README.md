# Лабораторная работа №17-18. Корутины на практике: Метеосводка

## Цель работы:

Изучить основы корутин в Kotlin для создания асинхронных операций в Androidприложениях. 
Научиться использовать launch, async, диспетчеры корутин и обрабатывать
длительные операции без блокировки UI-потока. Создать приложение "Weather Dashboard" с
имитацией загрузки данных о погоде из разных источников.
  
___
## Теоретическая часть

Представьте, что вы готовите ужин: пока варится суп, вы можете нарезать салат, а
пока запекается мясо — накрыть на стол. Вы не стоите и не ждёте, пока закончится
каждая операция. **Корутины** работают так же — они позволяют вашему приложению
выполнять несколько задач "параллельно", не блокируя основной поток (UI).

## Зачем нужны корутины?

* Загрузка данных из сети без "зависания" приложения
* Работа с базой данных
* Обработка файлов
* Простой и читаемый код

___

## Примеры кода

```kotlin
/ Обычная функция - выполняется сразу и полностью
fun fetchData(): String {
// Если здесь долгая операция - UI зависнет!
Thread.sleep(2000)
return "Data"
}

```
